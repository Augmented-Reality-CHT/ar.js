<static> addRegionListener(_onSuccess, _onError, _onSuccessListener, _onErrorListner)

Starts the region detection. The callback function specified in _onSuccessListener is called if a region was detected or was missing. A maximum of 10 region detection event listeners can be registered.
Parameters:
Name Type Description
_onSuccess Function Callback function called if region detection event listener registration was successful. Specify Function(AR.Native.Result). The argument passed from a native application upon successful completion is the following AR.Native.Result:
  • status {string} "OK"
  • value {string} Listener ID
_onError Function Callback function called if region detection event listener registration failed. Specify Function(AR.Native.Result) as the callback function. The argument passed from a native application upon failure is the following AR.Native.Result:
    When a parameter error occurs in a native application:
  • status {string} "INVALID_PARAMETER_EXCEPTION"
  • value {string} Error message
    When an attempt was made to register 11 or more region detection event listeners:
  • status {string} "INVALID_OPERATION_EXCEPTION"
  • value {string} Error message
_onSuccessListener Function Callback function called if the region detection state was changed. Specify Function(AR.Native.Result). The argument passed from a native application upon successful completion is the following AR.Native.Result:
  • status {string} "OK"
  • value {Object} Detection information
The detection information value is the following Object type:
  • status {boolean} true (detected), or false (missing)
  • uuid {string} UUID(Fomat is the hexadecimal notation in the case of 16-bit and 32 bit format. add the '-' in the case of 128-bit format. For example:123e4567-e89b-12d3-a456-426655440000)
_onErrorListner Function Callback function called if region detection failed. Specify Function(AR.Native.Result). The argument passed from a native application upon failure is the following AR.Native.Result:
    When a parameter error occurs in a native application:
  • status {string} "INVALID_PARAMETER_EXCEPTION"
  • value {string} Error message
Throws:
  • InvalidParameterError: When the argument is not the expected type, or is null/undefined
  • UnexpectedError: When generation of a request to a native application fails
  • InvalidStateError: When a native application call fails
Type
Error
Example
onSuccess = function(_result){
  alert("Listener ID : " + _result.getValue());
};

onError = function(_result){
  alert(_result.getStatus() + "\n" + _result.getValue());
};

onSuccessListener = function(_result){
  alert("Beacon with UUID "+ _result.getValue().uuid);
};

onErrorListener = function(_result){
  alert(_result.getStatus() + "\n" + _result.getValue());
};

AR.Bluetooth.addRegionListener(onSuccess, onError, onSuccessListener, onErrorListener);

AR Processing Server V1.1.1

<static> getCurrentRegions(_onSuccess, _onError)

Gets the currently detected regions list.
Parameters:
Name Type Description
_onSuccess Function Callback function called if region information acquisition was successful. Specify Function(AR.Native.Result) as the callback function. The argument passed from a native application upon successful completion is the following AR.Native.Result:
  • status {string} "OK"
  • value {Array.<Object>} Detection region information
A detection information value is an Object type array, with the following properties:
  • uuid {string} UUID(Fomat is the hexadecimal notation in the case of 16-bit and 32 bit format. add the '-' in the case of 128-bit format. For example:123e4567-e89b-12d3-a456-426655440000)
  • beacons {Array.<Object>} Beacon information
Beacon information is a object with following properties:
  • data {string} Beacon detail data (Advertisement packet)
  • rssi {number} Beacon received signal strength
_onError Function Callback function called if get current detected regions failed. Function(AR.Native.Result) The argument passed from a native application upon failure is the following AR.Native.Result:
    When a parameter error occurs in a native application:
  • status {string} "INVALID_PARAMETER_EXCEPTION"
  • value {string} Error message
Throws:
  • InvalidParameterError: When the argument is not the expected type, or is null/undefined
  • UnexpectedError: When generation of a request to a native application fails
  • InvalidStateError: When a native application call fails
Type
Error
Example
onSuccess = function(_result){
  var uuids = _result.getValue();
  for(var i=0; i<uuids.length; i++){
    var beaconList = "UUID: "+ uuids[i].uuid + "\n";
    var beacons = uuids[i].beacons;
    for (var j=0; j<beacons.length; j++) {
         beaconList +=   uuids[i].beacons[j].data + " : " + uuids[i].beacons[j].rssi + "\n";
    }
    alert(beaconList);
  }
};

onError = function(_result){
  alert(_result.getStatus() + "\n" + _result.getValue());
};
