<static> cacheUrlResource(_url, _onSuccess, _onError)

Save the resources of the specified URL to offline storage. There will be an error if the application is in offline or standalone mode. Only support http(s) Protocol. Use it to enable the offline storage . When the network communication is starting the onSuccess function will be called. If the communication failed or there is no resource under the URL, an error message popup dialog will appear.
Parameters:
Name Type Description
_url string resource URL
_onSuccess Function Callback function called if cache URL resource was successful. Specify Function(AR.Native.Result) as the callback function. The argument passed from a native application upon successful completion is the following AR.Native.Result:
  • status {string} "OK"
  • value {Object} null
_onError Function Callback function called if cache URL resource failed. Specify Function(AR.Native.Result) as the callback function. The argument passed from a native application upon failure is the following AR.Native.Result:
    When a parameter error occurs in a native application
  • status {string} "INVALID_PARAMETER_EXCEPTION"
  • value {string} Error message
    When an IO error occurs in a native application
  • status {string} "ARIO_EXCEPTION"
  • value {string} Error message
Throws:
  • InvalidParameterError: When the argument is not the expected type, or is null/undefined
  • UnexpectedError: When generation of a request to a native application fails
  • InvalidStateError: When a native application call fails
Type
Error
Example
onSuccess = function(_result){
  alert(_result.getStatus());
};

onError = function(_result){
  alert(_result.getStatus() + "\n" + _result.getValue());
};

var url = "http://192.0.2.0/sample.pdf";

AR.Data.cacheUrlResource(url, onSuccess, onError);

<static> getArServerData(_query, _isSuperReload, _onSuccess, _onError)

Sends a request to the AR processing server to acquire (GET) data. If offline storage has been set, query strings are used as a key when saving acquisition results to the offline storage. For the second and later data acquisition attempts using the same query, the data is obtained from the offline storage. If forced reading is specified, or if the results do not exist in the offline storage, the data will be acquired from the AR processing server.
Parameters:
Name Type Description
_query string Query string for the AR processing server API. The string length must be between 0 and 4096 characters.
_isSuperReload boolean Specifies whether to perform forced reading from the AR processing server without acquiring data from offline storage. Set to "true" to perform forced reading, or "false" otherwise.
_onSuccess Function Callback function called if data acquisition was successful. Specify Function(AR.Native.Result). The argument passed from a native application upon successful completion is the following AR.Native.Result:
  • status {string} "OK"
  • value {Object} AR processing server response
_onError Function Callback function called if data acquisition failed. Specify Function(AR.Native.Result). The argument passed when a native application fails is the following AR.Native.Result:
    When a parameter error occurs in a native application:
  • status {string} "INVALID_PARAMETER_EXCEPTION"
  • value {string} Error message
    When an IO error occurs in a native application:
  • status {string} "ARIO_EXCEPTION"
  • value {string} Error message
    When an HTTP exception occurs in a native application:
  • status {string} "AR_HTTP_EXCEPTION"
  • value {Object} Object type with the following properties:
    • status {number} HTTP response status code
    • statusText {string} HTTP response status string
    • responseText {string} HTTP response body string
Throws:
  • InvalidParameterError: When the argument is not the expected type, or null/undefined
  • UnexpectedError: When generation of a request to a native application fails
  • InvalidStateError: When a native application call fails
Type
Error
Example
onSuccess = function(_result){
  alert(_result.getStatus());
  var quads = _result.getValue();
  var records = quads.records;

      if(qvalues[j].floatValue !== null) quad += qvalues[j].floatValue + "\n";

    alert(quad);
  }
};

<static> setArServerInfo(_url, _id, _password, _onSuccess, _onError)
